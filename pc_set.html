tips

pyenv global system is for changing back to system python2. pyenv local ~~~ can set target python under currernt directory, so

cd your_workspace_for_ros
pyenv local system

may be useful to change python only for ros.

Or, instead,

pyenv global system
cd your_workspace_for_python3
pyenv local anaconda*

If you want to activate your own environment, you can use the following commands.

conda activate hoge
conda deactivate

If you want to use both pyenv and conda activate, I recommend to use pyenv mainly.

pyenv local anaconda*/envs/hoge

can change the virtual environment in the current directory to hoge. If pyenv cannot switch the virtual environment, you should deactivate, then change the env.

If (base) is always activated when launching new terminal, you can ignore its activation by

conda config --set auto_activate_base false
